# k400plus-fn-remapper（ピボット済み / BLEキーボードブリッジ）

## 概要

本プロジェクトは当初、Logitech K400 Plus のファンクションキー問題をハードウェアで解決することを目的としていました。

しかし検証の結果、

```
Fキー（例: F6）= 既に Win + D に変換済み
```

という仕様により、後段での復元は不可能と判断し、当初目的は中止しました。
私のようなLinuxの人はsolaarでおＫ。

---

## 方針転換（ここが本体）

K400対応を諦め、プロジェクトは以下に再定義しました：

```
有線キーボード → BLEキーボード変換ブリッジ
```

構成：

```
USBキーボード
  ↓
CH9350（USB→UART）
  ↓
ESP32
  ↓
BLE HID
```

---

## 現在の実装

### ✔ シンプルパイプライン

```
RawFrame
  ↓
Decode（STATE抽出）
  ↓
BLE送信（そのまま）
```

特徴：

* 状態管理なし
* 推測なし
* タイミング依存なし
* フラグ地獄なし

---

## 動作

```
✔ 有線キーボード入力をそのままBLEで再現
✔ 修飾キー（Ctrl / Alt / Win）正常動作
✔ 同時押し（6KRO）対応
✔ 遅延ほぼ無し
```

---

## なぜうまくいったか

K400と違い、通常のキーボードは

```
入力情報が欠損していない
```

ため

```
「解釈」ではなく「転送」で成立する
```

---

## K400失敗からの学び

```
・欠損した情報は復元できない
・推測ロジックは状態地獄を生む
・入力が正しければ設計はシンプルでいい
```

---

## 成果

* CH9350フレーム解析
* BLE HID（KeyReport）送信
* USB→UART→BLE パイプライン構築
* シンプル設計の有効性の確認
