# k400plus-fn-remapper

## 概要

`k400plus-fn-remapper` は、Logitech K400 Plus キーボードのファンクションキー問題をハードウェアで解決するプロジェクトです。

PC側にソフトウェアをインストールせず、外部デバイス（ESP32）でキー変換を行い、UnifyingレシーバをそのままBluetoothキーボードとして扱えるようにします。

---

## 特徴

* ソフトウェア不要（完全ハードウェア解決）
* Unifying Receiver → BLEキーボード変換
* 6キー同時押し（6KRO）対応
* Fキー問題の大部分を解消

---

## コンセプト

USB処理は専用IC（CH9350）にオフロードし、ESP32はシリアル受信とキー変換処理のみに専念します。

```
K400 Plus
  ↓ (2.4GHz)
Unifying Receiver
  ↓ (USB)
CH9350（USB→UART変換）
  ↓ (UART)
ESP32（DevKitC）
  ↓ (BLE HID)
PC
```


## 実装ポイント

### 1. 状態転送モデル

CH9350は内部でキー状態を管理しているため、ESP32側では

```
受信した状態 → そのままBLE送信
```

で基本的に動作します。

---

### 2. Fキー問題

K400のFキーは以下の2種類に分かれます：

| 種類          | 内容                                  |
| ----------- | ----------------------------------- |
| SPECIALフレーム | F1, F2, F5, F9〜F12                  |
| STATE混入     | F3, F4, F6〜F8（Alt+TabやWin+Dなどに展開済み） |

本プロジェクトでは：

* SPECIAL → 直接Fキーへ変換
* STATE混入 → パターンマッチで補正

---

### 3. 制約（重要）

以下は仕様上回避できません：

* `F6 (Win+D)` と通常の `Win+D` の完全な識別は不可能
* CH9350が入力を「マクロ展開」してしまうため情報が欠損する

そのため：

> 一部のFキーは通常ショートカットと競合する可能性があります

---

## CH9350 フレーム構造（要点）

### フレーム種別

* `57 AB 82 A3`：ハートビート（無視）
* `57 AB 88`：キーイベント

---

### 状態フレーム

```
57 AB 88 0B 11 m2 00 k0 k1 k2 k3 k4 k5
```

* 最大6キー同時押し
* modifierはビットフラグ

---

### 特殊フレーム

```
57 AB 88 08 31 ...
```

* 単発イベント
* Fキーやメディアキー

---

## 制限事項まとめ

* 一部Fキー（特にF6）は完全再現不可
* OSショートカットと競合する場合あり
* CH9350の仕様に依存
