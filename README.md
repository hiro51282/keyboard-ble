# k400plus-fn-remapper

## 概要

`k400plus-fn-remapper` は、Logitech K400 Plus キーボードのファンクションキー問題をハードウェアで解決するプロジェクト。

PC側にソフトウェアを入れず、外部デバイスでキー変換を行う。

---

## コンセプト

USB処理は専用ICにオフロードし、ESP32はシリアル受信と変換処理に専念する。

```
K400 Plus
  ↓ (2.4GHz)
Unifying Receiver
  ↓ (USB)
CH9350（USB→UART変換）
  ↓ (UART)
ESP32（DevKitC）
  ↓ (BLE HID)
PC
```

## CH9350 フレーム構造（解析メモ・暫定 → 準確定）

※ 観測ログに基づく準確定仕様（引き続き要検証）

### フレーム種別

* `57 AB 82 A3`：状態同期/ハートビート（入力無関係のため基本無視）
* `57 AB 88 ...`：キーイベント本体

### フレーム長

* **可変長**
* `buf[3]` は **ペイロード長（目安）** の可能性が高い

  * 例：`0x0B`（状態フレーム）, `0x08`（特殊フレーム）

---

### ① 状態フレーム（メイン）

```
57 AB 88 0B 11 m2 00 k0 k1 k2 k3 k4 k5 XX XX
```

| index | 内容                    |
| ----- | --------------------- |
| 0-2   | ヘッダ（57 AB 88）         |
| 3     | 長さ（0x0Bが多い）           |
| 4     | モード（通常は0x11）          |
| 5     | modifier（ビットフラグ）      |
| 6     | 予約（0x00）              |
| 7-12  | **keys[6]（同時押しキー配列）** |
| 末尾    | カウンタ/チェックサム（2バイト）     |

#### 特徴

* **押下状態のスナップショットを毎回送信**
* 最大6キー（USB HID Boot Protocol準拠）
* 押されていないスロットは `0x00`

#### 例（QWERTY同時押し）

```
1C
1C 15
1C 15 17
1C 15 17 14
1C 15 17 14 08
1C 15 17 14 08 1A
```

→ 押されているキーが配列として増減する

---

### ② 特殊フレーム（Fキーなど）

```
57 AB 88 08 31 ...
```

#### 特徴

* 単発イベント
* keys配列には乗らない
* メディアキー / Fキー / マクロ系

#### 挙動

* 状態フレームとは別に送信される
* OS側で直接処理される動作（例：Alt+Tabなど）

---

### modifier（m2）

ビットフラグ形式

| ビット  | キー       |
| ---- | -------- |
| 0x01 | Ctrl(L)  |
| 0x02 | Shift(L) |
| 0x04 | Alt(L)   |
| 0x08 | Win      |
| 0x10 | Ctrl(R)  |
| 0x20 | Shift(R) |
| 0x40 | Alt(R)   |

例：

```
0x0B = Ctrl + Shift + Win
```

---

### 重要ポイント（設計に直結）

* CH9350は**キー状態管理を内部で完結している**
* ESP32側で状態管理は不要
* 実装は「そのまま転送」で成立する

| index | 値           | 仮説                       |
| ----- | ----------- | ------------------------ |
| 0     | 0x57        | フレーム開始ヘッダ                |
| 1     | 0xAB        | 固定値？（要検証）                |
| 2     | 0x88        | フレーム種別（キーイベント）           |
| 3     | 0x08 / 0x0B | 長さ / 種別（要検証）             |
| 4     | 0x31 / 0x11 | モード（通常 / 修飾系）            |
| 5     | m2          | Modifierビット（Shift/Ctrl等） |
| 6     | k1          | 補助キー / Consumer系         |
| 7     | k2          | メインキー（HIDコード）            |

### メモ

* `k2` は通常キーのHID Usageに相当する可能性が高い
* `m2` はビットフラグで修飾キー状態を表現
* `k1` はメディアキーや特殊キーで使用される可能性あり
* `buf[3]` と `buf[4]` はイベント種別やモードに関与している可能性が高い
* KEYUPは `k1 = 0x00, k2 = 0x00` で表現されるケースが多い

## 修飾キー（Modifier）マッピング

| キー       | m1   | m2   | 備考                  |
| -------- | ---- | ---- | ------------------- |
| Shift（左） | 0x11 | 0x02 | 押下で2、離すと0           |
| Shift（右） | 0x11 | 0x20 |                     |
| Ctrl（左）  | 0x11 | 0x01 |                     |
| Ctrl（右）  | 0x11 | 0x10 |                     |
| Alt（左）   | 0x11 | 0x04 |                     |
| Alt（右）   | 0x11 | 0x40 |                     |
| Win      | 0x11 | 0x08 |                     |
| Fn       | -    | -    | CH9350に出力されない（内部処理） |

※ KEYDOWN: m2にビットが立つ
※ KEYUP: m2 = 0x00

## CH9350 生データ取得コード（ログ用）

```cpp
#include <Arduino.h>

HardwareSerial mySerial(2);

void setup()
{
  Serial.begin(115200);
  mySerial.begin(115200, SERIAL_8N1, 16, 17);
}

void loop()
{
  static uint8_t buf[64];
  static int idx = 0;

  while (mySerial.available())
  {
    uint8_t d = mySerial.read();

    // ===== フレーム開始検出 =====
    if (d == 0x57)
    {
      if (idx > 0)
      {
        // 前フレーム吐く（途中でも）
        Serial.print("FRM: ");
        for (int i = 0; i < idx; i++)
        {
          if (buf[i] < 0x10) Serial.print("0");
          Serial.print(buf[i], HEX);
          Serial.print(" ");
        }
        Serial.println();
      }
      idx = 0;
    }

    // バッファ格納
    if (idx < sizeof(buf))
    {
      buf[idx++] = d;
    }

    // ===== セーフティ（長すぎる場合吐く） =====
    if (idx >= 32)
    {
      Serial.print("OVF: ");
      for (int i = 0; i < idx; i++)
      {
        if (buf[i] < 0x10) Serial.print("0");
        Serial.print(buf[i], HEX);
        Serial.print(" ");
      }
      Serial.println();
      idx = 0;
    }
  }
}
```
